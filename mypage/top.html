研究内容

近赤外線を用いた唾液分泌量の推定について研究しようと考えています。
高齢者や口腔機能の低下している方は、口腔内の乾燥による食事中の誤嚥に気を付けなければなりません。 そこで、口内の状況を定量的に判断できるようなシステムを研究しようと考えました。
食事は毎日するものなので、継続して使用してもらうことが重要となります。 そのため、システムを使う際の手軽さを重視しています。
パルスオキシメーターなどで用いられる近赤外線をこめかみ部分に当てることでヘモグロビン信号を調べることができます。 ヘモグロビン信号は唾液分泌によって増減することが確認されています。 この信号を機械学習し、唾液分泌量を推定していきます。

現在は、先行研究を調査して研究室内で発表、デバイスの作成に取り掛かっています。
また、レンタルサーバを借り、Docker、NS-3、ROS2についての勉強も研究室で行っています。

「アプリケーション情報を用いたハンドオーバの検討」

ハンドオーバなどに起因する遅延や品質劣化を最小化する手法をアプリケーション主導のハンドオーバを導入し解決する手法を提案しました。

下図は既存のハンドオーバがどのように行われているのか確認したものです。UE(端末)が左上端から右下端に移動する際、1→2→3の順番でハンドオーバが行われています。 ハンドオーバの回数が増えると、情報の受け渡しも増えるので動作が不安定になっていきます。

1→3のように進行方向に存在する基地局につなげることが理想です。そのため、位置情報や目的地などのアプリケーション情報を用いてハンドオーバ回数を減らす手法を提案しました。

横スクロールアクションゲーム

大学の授業で作成したものです。

チーム開発（3人）
Java
GUI

キーボード操作で遊ぶ横スクロール2Dアクションゲームを作成しました。

キャラクター、ブロックはテーマを決めて自作しました。五つのステージを用意し、それぞれにあったBGMを お借りして流しています。BGMとSEはオプション画面から音量調節を行うことができます。 プレイ画面では一時停止、ゲームオーバー、ステージクリア画面を作成し、遊びやすいデ ザインにしています。

自動作曲

大学の授業で作成したものです。

MIDIプログラミングによる自動作曲
C++

作品を作り始めた時期に、東京都のコロナ新規感染者数が増え、ニュースになっていました。
感染者数推移のグラフを見ていたところ、スペクトル解析した際のグラフに見え、音楽にできるのではないかと考えたのでこの作品を制作しました。

感染者数が増えると音が大きくなり、高くなっていきます。ホラー風のものにしたかったので、「エクソシスト」や「Saw」のテーマ曲を参考にして、ベルの音で作ることにしました。

感染者数のデータを変更、増やすなどすれば自動作曲で曲が変わるため、時期によっては全然違うものになります。
